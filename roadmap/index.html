Roadmap

The following represents current view of Alien4Cloud development team of its product development cycle and future directions. It is intended for information purposes only, and should not be interpreted as a commitment, though we do our best to reach the dates and features set mentioned below.

Premium features, support and certifications are available under a subscription, delivered by FastConnect, a Bull, Atos technologies subsidiary. Contact us for more details.

We deliver major releases every 4 month. Please note that we also deliver intermediate sprint releases in Alien4Cloud GitHub repo, every 3 weeks, in order to get feedback on features while still in development.

  • May 2017 v 1.4.0

    Core

    • Manage topologies versions: ability to have several topologies for a specific Application version. Allows to adapt for a same application version to different deployment architectures (HA, LB, security, …) per environment.
    • Fine grained access rights on locations and resources within a location
    • Services management
      • Declare already provisioned resource as a service accessible by Alien4Cloud application resources
      • Expose a deployed application as a service
    • Advanced post-deployment updates, upgrades & sync
    • Security improvements on runtime within Cloudify 4.0 – Premium
    • End to end HA support improvement when leveraging Cloudify runtime orchestrator
    • Dev/Ops/User experience improvements
      • ease Types and Topologies development and tests
      • leverage Docker to improve instanciation time when developing/testing
      • improve logs screens usability
      • Jenkins plug-in, bringing Continuous Delivery to Jenkins users

    Orchestrators certifications

    • Cloudify v4.0

    Infrastructure support

    • Ansible plug-in improvements for more native support 
    • Docker
      • On-demand hybrid deployment through Cloudify integration & user defined matching
    • Certification for following infrastructures/IaaS
      • Amazon Web Services
      • Microsoft Azure
      • VMWare VSphere
      • Openstack
      • Host-pool (physical machines) 

    TOSCA conformance

    • Small enhancements to improve TOSCA 1.0 YML conformance
    • Support for alien 4 cloud 1.2 and 1.3 DSL

    Additional Premium features

    • v1.3 to v1.4 Migration tools
  • To be defined Future
